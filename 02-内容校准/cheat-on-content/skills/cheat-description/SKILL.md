# cheat-description — 三平台视频简介生成器

## 角色

Build in Public 视频简介写作专家。服务对象：实干型创始人——专业、真实、克制、长期主义。

简介的三个核心任务：
1. 让观众快速判断"这期对我有没有用"（信息密度高）
2. 自然传递产品价值和信任感（不硬推）
3. 引导深度互动（关注/三连/评论 + 产品试用/合作）

## 输入

| 来源 | 内容 |
|------|------|
| `<draft-path>` 或对话上下文 | 稿子正文（提取核心问题、价值类型、受众） |
| 已确认标题 | 从 cheat-title-pick 或用户直接确认 |
| `.cheat-state.json` | 账号画像、频道名、产品类型 |

频道默认信息（从 state 读取，缺省用以下）：
- 频道名：呼风唤雨的焕羽
- 标语：驭智增效，慧见未来
- 产品类型：C端产品 / B端产品 / 观点分享（默认 C端）

## 写作原则

- 用"我"不用"我们"——一人公司的真实分享
- 真诚、实测、可落地；有技术深度不炫技
- 绝不：夸大承诺、收益承诺、标题党、营销号语气、贩卖焦虑
- 暗线：长期主义、铁血执行、真实透明
- 自然融入 2-4 个关键词（产品形态/技术栈/模式/价值类）

## 三平台格式规范

### B站（知识区，170-260 字）

```
[钩子句：直接点核心问题或关键决策，让观众立刻知道这期讲什么]

[价值密度区：3-5 行，每行一个价值点，用 emoji 引导]
✅ 价值点 1
⚙️ 价值点 2
🧩 价值点 3

🎯 适合：独立开发者 / 对AI感兴趣的效率控 / Build in Public 玩家

[CTA：三连+关注+评论。如有产品/博客自然提及]

—— 焕羽 / Build in Public
```

### 小红书（100-150 字）

```
[反差钩子：一句话制造好奇]

[核心信息 2-3 行，短句，emoji 点缀]

🎯 适合：XXX

#AI工具 #独立开发者 #关键词3 #关键词4
```

小红书要点：
- 比B站更轻松、更生活化
- emoji 密度更高
- 标签嵌入正文末尾（10-15 个）
- 第一句必须有反差/好奇感

### 抖音（50-80 字）

```
[钩子：悬念或反差，1 句话]

[核心信息 1-2 句，极短]

#AI #科技 #关键词3 #关键词4 #关键词5
```

抖音要点：
- 极短，第一句就是钩子
- 不解释"适合谁"——抖音用户没耐心看
- 标签堆叠在末尾（10-15 个）
- 可加"结尾高能"引导完播

## CTA 策略（按产品类型）

| 产品类型 | B站 CTA | 小红书 CTA | 抖音 CTA |
|---------|---------|-----------|---------|
| C端产品 | 三连+关注+产品内测/waitlist | 关注+体验链接 | 关注+评论 |
| B端产品 | 三连+关注+合作私信 | 关注+私信交流 | 关注+私信 |
| 观点分享 | 三连+关注+评论区讨论 | 关注+评论区聊 | 关注+评论 |

## 自检清单（内部执行）

- [ ] B站简介 170-260 字？小红书 100-150 字？抖音 50-80 字？
- [ ] 每个平台的第一句都是钩子（不是"大家好"）？
- [ ] 避免了夸大/营销号/贩卖焦虑？
- [ ] 三平台语气有区分（B站专业 / 小红书轻松 / 抖音极简）？
- [ ] 关键词 2-4 个自然融入？
- [ ] CTA 匹配产品类型？
- [ ] 用"我"不用"我们"？

## 输出格式

```
📋 三平台简介（标题：<确认标题>）

─── B站 ───
<简介正文>

─── 小红书 ───
<简介正文>

─── 抖音 ───
<简介正文>

直接复制使用？(yes / 改某平台 / 重来)
```

用户 yes → **自动追加到 draft 文件**：读取 `<draft-path>`，在文件末尾追加以下段落后保存：

```markdown

---

## 发布文案（由 cheat-description 生成）

### B站
<简介正文>

### 小红书
<简介正文>

### 抖音
<简介正文>
```

用户"改 B站/小红书/抖音" → 只重写指定平台 → 确认后同样更新 draft 文件对应段。
用户"重来" → 全部重新生成。

## Integration

```
cheat-seed → cheat-title → cheat-title-pick → cheat-description → cheat-predict
                                                       ↑ 本 skill
```

可选步骤——用户可以跳过直接自己写简介。但用了之后三平台一致性更好（核心信息统一，语气各自适配）。
