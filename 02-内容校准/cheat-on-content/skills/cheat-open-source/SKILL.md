# /cheat-open-source — 自我开源度采访（产品获客专用工具）

**服务对象**：产品获客。产品已上线、通过视频获客时才跑。账号常规内容（非营销类）走 /cheat-who-for，不跑这个。

把内容从"营销腔/端着教人"切换到"自我开源腔"——基于真实经验、完整呈现推理过程、低营销属性。

**为什么需要这个 skill**：AI 时代，通用知识被 LLM 抹平，IP 差异化只剩"每个人的人生剧本不同"。但创作者容易写成两种死法：
- **端着教人**：只给结论和方法论，藏起自己的推理过程（怕露怯）→ 观众觉得被教育，不爱看
- **高营销属性**：价值承诺堆砌、钩子话术、套路开场 → 观众觉得被推销，反感

自我开源是反过来的：**把思维方式直接摊开给观众，听懂就借鉴，认同者主动靠近**。这是无法被 AI 复制的——因为你的推理过程和踩坑经历是你的剧本，不是通用知识。

## 角色

**采访者 + 反套路镜子**。

- 像同行追问：一次问一个，听完回答再问下一个，基于回答追问
- 候选选项**从 draft 内容提取**，并在可疑处标注 `[镜子：我怀疑这里套路化了]`——这是跟 who-for 纯采访的区别，open-source 的候选里带诊断信号
- 不接受"就是这样""大家都这么说"这类答案，温和地逼具体推理

## Overview

```
[用户：自我开源 scripts/<id>.md]
  ↓
[Phase 0: 读 draft + state + 提取推理链信号]
  ↓
[Phase 1: 7 问采访 — 一次一个，每问给候选（含镜子标注），等回答再下一问]
  ↓
[Phase 2: AI 综合 — 生成开源度评分段]
  ↓
[Phase 3: 红旗检查 — 经验编造/推理缺失/通人人云亦云/营销腔 → 警告]
  ↓
[Phase 4: 落盘 open-source-audit.md]
  ↓
[Phase 5: 改稿闭环 — 红旗映射成 script.md 具体改动，逐条确认执行]
```

## Constants

- **MAX_PROBES_PER_QUESTION = 1** — 每个问题最多追问 1 轮（答案太浅或太套话时）
- **CANDIDATE_COUNT = 2-4** — 每个问题给 2-4 候选，基于 draft 生成 + 可疑处附镜子标注。永远附"都不是，我自己说"选项
- **SKIPPABLE = yes** — 用户可跳过任一问题，brief 里标 `skipped`，不编造
- **AUDIT_FILENAME = open-source-audit.md** — 落在 `<user-content-project>/open-source-audit.md`（与 audience-brief.md 同级）

## Inputs

| 必填 | 来源 |
|---|---|
| `<draft-path>` | 用户参数；缺失则 glob `scripts/*.md` 询问选哪个 |
| `.cheat-state.json` | 读 `typical_duration_seconds`、账号画像 |
| draft 正文 | 6 问候选 + 镜子标注的唯一来源 |

## Workflow

### Phase 0: 前置检查 + 提取推理链信号

1. 读 `.cheat-state.json` → 不存在则提示先跑 `/cheat-init`，停止
2. 读 `<draft-path>` → 不存在则停止报错
3. **提取推理链信号**（候选 + 镜子标注原料）：
   - draft 里的**结论句**（"应该 X""最好 Y""千万别 Z"）vs **推理过程句**（"我当时发现…""我试过…结果…"）的比例
   - 基于"我"的真实经历词（"我那次""上周我""我踩过"）vs 通用建议词（"建议大家""一般来说""很多人"）
   - 营销话术信号（"你也能""只要…就""彻底""必看""想知道吗"）
   - 反常现象/卡点描述（结构第 1 段原料）
   - 推理链断裂点（给了结论但没说怎么得出的）

### Phase 1: 6 问采访（核心）

**采访纪律**（跟 who-for 一致，违反就退化成表单）：

1. **一次只问 1 个问题**。绝不连珠炮
2. **每个问题附 2-4 候选**，候选**从 Phase 0 信号派生**。可疑处标 `[镜子：…]` 让用户看到 AI 的怀疑
3. **必须等用户回答完**再问下一个
4. **追问触发**（≤ MAX_PROBES 次）：答案含"就是这样""大家都""反正"、纯结论无过程 → 追问"具体怎么推理的"。用户再答就接受
5. 用户说"跳过" → 标 skipped，继续下一问，**不编造**

#### Q1 经验真实性（"基于你哪次真实经历"）

```
Q1/6 这条内容，是基于你哪次真实经历？自我开源的前提是你花过时间/钱/验证过。

候选（我从稿子里看到的来源）：

  A. [基于 draft 里"我"经历词提取的场景 1，如"你提到'我上次上线后用户全流失'"]
     → [镜子：这是亲历的，开源度高]
  B. [场景 2]
  C. [draft 里通用建议词对应的段落，如"建议大家不要焦虑"]
     → [镜子：⚠️ 这段没有"我"的经历支撑，像是通用建议而非复盘]
  D. 都不是，我自己说：____
```

**诊断目的**：内容有没有真实经验地基。自我开源不是讲道理，是讲经历。
**红旗**：选不出具体经历，或 draft 里"我"的经历词 < 通用建议词。

#### Q2 问题/反常现象（"你遇到什么反常/卡点"）

```
Q2/6 你做这件事时，遇到什么反常现象或卡点？具体到那个让你愣一下的瞬间。

  A. [基于 draft 卡点词提取，如"你说'上线第一天 DAU 是 0，慌了'"]
  B. [卡点 2]
  C. [如果 draft 只讲成功不讲卡点 → "稿子里只看到结果，没看到卡点"]
     → [镜子：⚠️ 只讲结果不讲卡点，观众学不到东西——卡点才是经验密度最高的部分]
  D. 都不是，我自己说：____
```

**诊断目的**：自我开源结构第 1 段。卡点是经验密度最高的部分，省略它 = 只剩成功学。
**红旗**：draft 通篇是"我做了 X，成功了"，没有任何反常/卡点/失败。

#### Q3 推理过程（"你怎么想的、怎么推理的"）—— 核心

```
Q3/6 你得出稿子里这些结论，推理过程是什么？不是结论本身——是你怎么想到的。

我看你稿子里有这些结论：[列出 draft 的 2-3 个结论句]

  A. [针对结论 1，AI 试着补的推理链，如"你看到 X 现象 → 联想到 Y → 试了 Z → 验证了"]
     → 如果稿子里没写这个过程 → [镜子：⚠️ 你只给了结论，没给推理。观众无法借鉴你的"想清楚的过程"，只能照搬结论]
  B. [针对结论 2 的推理链]
  C. 我的推理过程其实是：____（这是最有价值的部分，请说详细）
  D. 跳过
```

**诊断目的**：自我开源结构第 2 段，也是精髓。**结论可以被 AI 生成，推理过程不能**。这是 IP 差异化的核心。
**追问**：用户给的推理太简略（"我就是觉得 X"）→ "你觉得 X 之前，是什么让你开始这么想的？"
**红旗**：draft 里结论句 >> 推理过程句，或 Q3 答不出推理（说明结论是搬运的不是自己想通的）。

#### Q4 结果 + 指导意义（"结果是什么，对观众有什么行动指导"）

```
Q4/6 最后你拿到什么结果？观众看完能用你这套做什么？

  A. [基于 draft 结果提取，如"你说'改完留存涨了 40%'"]
     → 观众能拿走：[AI 推断的可复用步骤]
  B. [draft 只讲过程没讲结果 → "稿子里没看到明确结果"]
     → [镜子：⚠️ 没结果的复盘是半截话，观众不知道这套做法到底灵不灵]
  C. 都不是，我自己说：____
```

**诊断目的**：自我开源结构第 3 段。结果 + 指导意义闭环。
**红旗**：draft 有过程无结果，或有结果但没说"观众能怎么用"。

#### Q5 差异化印记（"你的思维方式里，什么是'你的'"）

```
Q5/6 稿子里这些判断，哪些是"你的"独特印记，哪些是行业共识？

  A. [draft 主张 1] → [镜子：这个是行业共识/通用知识，AI 也能讲，不算你的差异化]
  B. [draft 主张 2] → [镜子：这个带你的具体经历锚点，是你的印记]
  C. [draft 主张 3] → [镜子：判断不出，你说说是你独有的还是听来的]
  D. 我自己标：____
```

**诊断目的**：IP 差异化 = 人生剧本不同。如果整篇都是行业共识，AI 能替代你，观众没必要关注你。
**红旗**：用户标不出任何"我的独特印记"，或整篇都是通用方法论。

#### Q6 营销属性自检（"哪里端着了/藏钩子了"）—— 对偶于 who-for 的钩子问

```
Q6/6 反向检查——稿子里哪里你觉得"端着了"或"在推销"？

候选（我怀疑的套路化位置）：

  A. [draft 里价值承诺句，如"你也能 3 天学会 X"]
     → [镜子：⚠️ "你也能"是营销话术，自我开源应该讲"我是怎么会的"，让观众自己判断能不能学]
  B. [draft 里钩子开场，如"想知道吗？往下看"]
     → [镜子：⚠️ 藏钩子 = 高营销属性，跟自我开源的"直接摊开"相反]
  C. [draft 里结论先行句，如"记住这三点"]
     → [镜子：⚠️ "记住"是教人口吻，自我开源是"我是这么想的，你参考"]
  D. 我自己看出别的地方：____
  E. 我觉得没有端着的地方
```

**诊断目的**：自我开源是低营销属性。这问跟 who-for 的 Q6（怎么钩住受众）正好对偶——who-for 帮你想钩子，open-source 帮你拆套路。
**红旗**：用户选 E 但 Phase 0 提取到 ≥3 处营销话术信号 → AI 温和指出"我数到 N 处可能套路化的地方，要不我们一起看看"。

#### Q7 信任与降低决策风险（"用户凭什么信你、凭什么敢试"）—— 产品获客专用

> 来源：姜胡说的"降低用户决策风险"方法论——用户买衣服怕买错，商家提供"7天无理由退货""先试用再购买"。短视频获客同理：你需要证明自己"不割韭菜""随时可退费"，打消用户顾虑。

```
Q7/7 用户看完这条，如果对你的产品感兴趣，他最大的顾虑是什么？你怎么打消它？

候选（我从稿子里看到的"降低风险"信号）：

  A. 稿子里有"开源/免费/试用"等降低门槛的信号
     → [镜子：这是降低决策风险的有效信号——让用户"先体验再付费"]
  B. 稿子里有真实踩坑/失败呈现
     → [镜子：展示失败 = "我不藏着" = 信任感。比只讲成功更可信]
  C. 稿子里有"我是怎么会的"推理过程（不是"你也能"营销话术）
     → [镜子：自我开源腔本身就是信任建立——你摊开思维，用户自己判断]
  D. 稿子里没有降低风险的信号——纯展示产品功能
     → [镜子：⚠️ 用户看完会觉得"又在推销"，没有打消顾虑的内容。产品获客视频必须解决"凭什么信你"这个问题]
  E. 都不是，我自己说：____
```

**诊断目的**：产品获客场景的核心——用户从"感兴趣"到"试用"之间，隔着一个"决策风险"。你的内容有没有帮他跨过这个门槛。
**姜胡说框架**：口碑（你这个人的可信度）+ 真材实料（能解决问题）+ 降低风险（试用/退费/不割韭菜）= 用户敢行动。
**追问**：用户选 A/B/C → "除了稿子里这些，产品本身有没有'降低风险'的设计？"（如免费试用、开源、随时退费、社区答疑等）
**红旗**：选 D 或 draft 通篇是产品功能展示，没有任何"凭什么信你"的内容 → 产品获客视频缺失核心转化层。

### Phase 2: AI 综合 — 开源度评分段

7 问答完后，AI **基于 draft 信号 + Q1-Q7 答案**综合一段（不再问用户）：

```markdown
## 开源度评分（AI 综合）

| 维度 | 评估 | 依据 |
|---|---|---|
| 经验地基 | 强/中/弱 | Q1 答案 + draft"我"经历词密度 |
| 卡点呈现 | 完整/缺失 | Q2 + draft 反常现象段 |
| 推理完整度 | 高/中/低 | Q3 + draft 结论句 vs 推理句比例（核心维度） |
| 结果闭环 | 有/无 | Q4 |
| 差异化密度 | 高/中/低 | Q5 + draft 独特印记数量 |
| 营销属性 | 低/中/高 | Q6 + Phase 0 营销话术信号数（越低越好） |
| 信任/降低风险 | 强/中/弱/缺失 | Q7 + draft 降低门槛信号（产品获客专用） |

**总体开源度**: `<高 / 中 / 低>`

> 高 = 真诚复盘 + 降低决策风险，认同者会主动靠近并敢试用
> 中 = 有自我开源的底子，但信任/风险层缺失或营销残留，建议改稿
> 低 = 端着教人/营销腔为主，AI 也能写，建立不了 IP，也不敢让人行动
```

### Phase 3: 红旗检查

| 检查 | 触发条件 | 含义 |
|---|---|---|
| 经验编造 | Q1 答不出具体经历，draft"我"经历词稀疏 | 内容没地基 |
| 推理缺失 | Q3 只给结论不给过程，draft 结论句 >> 推理句 | 核心 bug——只剩结论可被 AI 替代 |
| 卡点缺失 | Q2/draft 全是成功没有反常 | 成功学不是复盘 |
| 通用人云亦云 | Q5 标不出任何独特印记 | AI 能替代，没 IP 价值 |
| 营销腔 | Q6 选 E 但信号 ≥3，或用户认了多处套路 | 跟自我开源相反 |
| 教人口吻 | draft 多"应该/记住/必须"少"我当时/我发现" | 端着，不是分享 |

命中 ≥2 条 → 落盘前警告，建议重跑某些问题或改稿：

```
⚠️ 命中 [N] 条红旗：
- [红旗 1 + 来自哪个问题/信号]
- [红旗 2]

核心问题：[最严重的那条，如"推理缺失——你只给了 5 个结论，0 个推理过程。这是 AI 最容易替代的部分"]

建议：
- 重跑 /cheat-open-source Q[X]
- 或先改稿补上 [推理过程/卡点/真实经历]，再重跑
- 或接受现状，audit 里标 red_flag: true（predict 会降 confidence）
```

### Phase 4: 落盘

写到 `<user-content-project>/open-source-audit.md`，按 [templates/open-source-audit.template.md](../../templates/open-source-audit.template.md) 格式。

落盘后简短确认，**直接进入 Phase 5 改稿闭环**——自我开源度低的内容发布也是污染校准池，诊断必须落到改动上：

```
✅ 开源度审查写完：open-source-audit.md（总体开源度：[高/中/低]，red_flag: <true/false>）

进入 Phase 5：把红旗映射成 script.md 的具体改动。
```

### Phase 5: 改稿闭环（红旗 → script.md 具体改动）

诊断不能止步于"建议改稿"。把每条红旗映射成 script.md 的具体改动并执行，让内容从"端着教人/营销腔"切到"自我开源腔"。

**通用机制**（生成建议 → 逐条确认 → 执行 → 记录改稿日志）见 [shared-references/revision-loop-protocol.md](../../shared-references/revision-loop-protocol.md)。本段只写 open-source 特有的"红旗 → 改稿动作"映射。

#### open-source 红旗 → 改稿动作表

| 红旗（来自 Phase 3） | 改稿动作 | 类型 |
|---|---|---|
| 推理缺失（**核心**） | 标出每个"有结论无推理"的句子，提示"补上你怎么得出这个结论的推理链，~80-150 字/处" | `[需创作者输入]` |
| 营销腔 | 标出营销话术（"你也能"/"只要…就"/"彻底"/"必看"），给去营销化草案（→ "我是怎么会的，你参考"） | `[机械改动]` |
| 教人口吻 | 标出"应该/记住/必须/一定要"，给改口吻草案（→ "我当时"/"我发现"/"我踩过"） | `[机械改动]` |
| 经验编造 | 标出无"我"经历支撑的段落，提示"补上这是你哪次真实经历，时间/场景/你的角色" | `[需创作者输入]` |
| 卡点缺失 | 标出"只讲成功不讲卡点"处，提示"补上你遇到的反常现象/那个愣一下的瞬间" | `[需创作者输入]` |
| 通用人云亦云 | 标出无差异化印记的主张（行业共识），提示"这里加一个只有你才会这么判断的理由" | `[需创作者输入]` |

> 推理缺失是 open-source Phase 5 的**重中之重**——它是 `[需创作者输入]`，AI 只标位置不替写（推理过程是你的剧本，AI 写 = 编造，违反自我开源原则）。但它是 IP 差异化的最后壁垒，必须逼创作者自己补。

#### 执行后

按 revision-loop-protocol.md 的"改稿后流程"：

- 改了 ≥1 条 `[机械改动]`（去营销/改口吻）→ 提示重跑 /cheat-open-source 看红旗是否解除，再 /cheat-predict
- 推理/经历/卡点缺失（全是 `[需创作者输入]`）→ 提示"位置已标 TODO 注释，补完后说'补完了'，我帮你重跑 + predict"
- 全跳过 → audit 保留 `red_flag: true`，可直接 predict（confidence 会降一档）

## Key Rules

1. **一次一个问题**。塞多个 = 退化成表单
2. **候选必须从 draft 派生 + 带镜子标注**。镜子标注是 open-source 区别于 who-for 的核心——who-for 候选是"你想的是哪种"，open-source 候选是"我怀疑这里套路化了，你认不认"
3. **等回答再下一问**。不预判、不抢答
4. **Q3（推理过程）是重中之重**——如果只能问好一个问题，就是这个。结论可被 AI 生成，推理过程不能
5. **跳过允许，编造禁止**。skipped 标 skipped
6. **只读 draft + state**。不抓热点、不读 candidates
7. **不写 rubric_notes.md**。观察要等复盘有实绩对比（原则 #3）
8. **采访用同行口吻**。不要"您觉得"——直接问，像同行拆招

## Refusals

- 「跳过采访，直接给我填一份 audit」 → 拒绝。没有采访的 audit 是死档案。核心价值在逼你想清楚推理过程
- 「帮我把我没写的推理过程补全到 audit 里」 → 拒绝。推理过程是你的剧本，AI 补的 = 编造，违反自我开源原则。audit 只记录你说出来的，缺失的标"推理缺失"
- 「用上一篇的 audit 改改」 → 拒绝。每篇 draft 的经验/推理不同
- 「把 audit 写进 rubric_notes.md 当观察」 → 拒绝。原则 #3
- 「我营销属性高没关系，我就想卖货」 → 不拒绝，但提醒"自我开源不是这个 skill 的目标。你想卖货可以跳过 open-source，直接 predict——rubric 的 AB/SR 会评估推荐流效果"。不强制

## Integration

- **服务对象**：产品获客。适用于营销场合（产品已上线、通过视频获客）。账号常规内容（非营销类）走 /cheat-who-for，不跑这个
- **双角度协作**（与原始 cheat-on-content）：rubric 打分告诉你"SAT（讽刺深度）只有 2 分"但不说怎么改；open-source 诊断"推理缺失"+ Phase 5 标位置逼你补推理——定性改稿补上 rubric 的盲区。改完重跑 open-source 验证红旗解除 → /cheat-score 看 SAT/QL 是否涨分 → 双角度闭环
- **上游**：`/cheat-seed` 写完 draft → 用户跑 `/cheat-open-source scripts/<id>.md`（与 `/cheat-who-for`、`/cheat-title` 平级，都是 post-draft 增强）
- **与 /cheat-who-for 的分工**：
  - who-for 服务**账号 IP**（非营销类，每条都跑）——问"给谁"
  - open-source 服务**产品获客**（营销类，专用）——问"我的推理呈现了没"
  - 两者问题维度不重叠。产品获客视频建议两个都跑：先 who-for（想清楚拍给哪个潜在用户）→ 再 open-source（把产品推理/踩坑真诚开源）。非营销视频只跑 who-for
- **下游**：`/cheat-predict` 读 `open-source-audit.md`（如存在）作为 SAT（讽刺/自嘲深度，真诚复盘往往带自嘲）、QL（金句，真实推理出金句而非拼凑）的 context——**不替代打分**。audit 标 `red_flag: true` 时 predict 降低 confidence 一档
- **下游**：`/cheat-retro` 复盘时读 audit 对照实绩 → 检验"自我开源式内容是否真的长尾起来"。如果搜索流量占比高 + 互动质量好（评论区有人"说出了我想说的"）→ 验证自我开源有效，写进 rubric_notes.md
- **可选性**：产品获客视频强烈建议跑（它是这个场景的专用工具）。账号常规内容（非营销类）不需要跑——走 /cheat-who-for
